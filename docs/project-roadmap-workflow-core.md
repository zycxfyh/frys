# 🚀 frys 工作流系统 - 核心定位发展路线图

## 🎯 项目核心定位

**frys 是一个专注于工作流自动化的开源引擎，通过插件生态和平台服务，打造工作流领域的核心基础设施。**

### 核心原则

- 🎯 **工作流优先**：一切功能围绕工作流引擎展开
- 🧩 **插件扩展**：通过插件实现功能扩展，而非内置
- 🌐 **平台服务**：基于工作流核心提供增值服务
- 📈 **渐进发展**：按阶段有序推进，避免功能蔓延

### 产品定位

```
frys Core (工作流引擎) ← 核心产品
├── frys Plugins (插件生态) ← 扩展能力
├── frys Platform (管理平台) ← 用户界面
└── frys Cloud (商业服务) ← 商业化产品
```

## 🔧 技术架构

### 核心架构

```
┌─────────────────┐
│   frys Core     │ ← 工作流引擎 (核心)
│ ├─ DSL Parser   │
│ ├─ Executor     │
│ ├─ State Mgmt   │
│ └─ Event Bus    │
└─────────────────┘
         │
    ┌────▼────┐
    │ Plugins │ ← 插件系统 (扩展)
    │ ├─ AI    │
    │ ├─ DB    │
    │ └─ API   │
    └─────────┘
         │
    ┌────▼────┐
    │ Platform│ ← 管理平台 (界面)
    │ ├─ Web UI│
    │ ├─ CLI   │
    │ └─ API   │
    └─────────┘
         │
    ┌────▼────┐
    │ Cloud   │ ← 商业服务 (商业化)
    │ ├─ SaaS │
    │ └─ Ent   │
    └─────────┘
```

### 技术栈

- **核心引擎**：Node.js + TypeScript
- **执行引擎**：自定义工作流执行器
- **插件系统**：ESM模块 + 沙箱机制
- **Web界面**：React + 拖拽库
- **数据存储**：PostgreSQL + Redis
- **部署方式**：Docker + Kubernetes
